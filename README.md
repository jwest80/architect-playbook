# Architect Development Playbook

This playbook is a personal architecture workspace for the first 12 months of moving from Lead Developer to Architect. It captures learning progress, reusable architecture recipes, cloud patterns, decision records, diagrams, and small proofs of concept that turn study into usable artifacts.

The guiding principle is simple: learn by building reusable architecture artifacts. Each topic should produce something practical, such as a decision record, recipe, diagram, review checklist, proof of concept, or reference note that can be reused in real work.

## Working Rhythm

1. Pick a learning focus from the 12-month program.
2. Build a small proof of concept or research note.
3. Convert the learning into a reusable recipe, decision record, diagram, or template.
4. Record progress, open questions, and next steps.
5. Revisit artifacts as experience grows.
